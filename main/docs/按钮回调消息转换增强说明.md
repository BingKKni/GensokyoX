# 按钮回调消息转换增强功能说明

## 📄 功能概述

当启用 `global_interaction_to_message` 配置时，系统会将按钮和表态回调转化为消息。现在增强了这个功能，**自动在消息中包含QQ平台下发的原始交互ID**。

## 🔧 配置说明

**配置项**：`global_interaction_to_message`
- **位置**：`config.yml` 或配置文件中
- **类型**：`boolean`
- **默认值**：`false`
- **说明**：是否将按钮和表态回调转化为消息

## 🆕 新增字段

在以下消息结构体中新增了 `interaction_id` 字段：

### 📋 支持的消息类型

| 消息类型 | 结构体 | 适用场景 |
|---------|--------|----------|
| 群消息 | `OnebotGroupMessage` | 群聊中的按钮回调 |
| 群消息（字符串版本） | `OnebotGroupMessageS` | 字符串模式的群聊按钮回调 |
| 私聊消息 | `OnebotPrivateMessage` | 私聊中的按钮回调 |
| 频道消息 | `OnebotChannelMessage` | 频道中的按钮回调 |

### 📝 字段定义

```json
{
    "interaction_id": "1a2b3c4d5e6f7890"  // QQ平台下发的原始交互ID
}
```

- **字段名**：`interaction_id`
- **类型**：`string`
- **来源**：QQ平台回调事件中的 `data.ID` 字段
- **可选性**：可选字段（`omitempty`）
- **设置条件**：仅当启用 `global_interaction_to_message` 时才设置

## 🔄 工作原理

### 启用转换时的流程

1. **用户点击按钮** → 触发回调事件
2. **检查配置** → 如果 `global_interaction_to_message = true`
3. **转换消息** → 将回调转换为相应的消息格式
4. **设置ID** → 自动设置 `interaction_id = data.ID`
5. **发送消息** → 向应用端发送包含交互ID的消息

### 禁用转换时的行为

- **配置**：`global_interaction_to_message = false`（默认）
- **行为**：发送 `OnebotInteractionNotice` 类型的通知
- **包含内容**：完整的原始回调数据（`data` 字段）

## 💡 使用建议

### 🎯 适用场景

1. **需要消息格式**：希望按钮回调以消息形式处理
2. **需要交互ID**：需要获取原始交互ID进行后续操作
3. **统一处理**：希望按钮回调和普通消息使用相同的处理逻辑

### ⚙️ 配置建议

```yaml
# 推荐配置：启用消息转换
global_interaction_to_message: true

# 配合使用的其他配置
array: false                    # 是否使用数组格式
native_ob11: false             # 是否使用原生OneBot11格式
```

### 🔧 开发建议

1. **检查字段存在**：处理消息时检查 `interaction_id` 是否存在
2. **区分消息类型**：通过 `real_message_type` 字段识别是否为交互消息
3. **回应处理**：可以使用 `interaction_id` 进行手动回应

## ⚠️ 注意事项

1. **配置依赖**：只有启用 `global_interaction_to_message` 时才会包含 `interaction_id`
2. **向下兼容**：新字段为可选字段，不影响现有功能
3. **ID唯一性**：每个交互ID只能使用一次进行回应
4. **字段类型**：`interaction_id` 始终为字符串类型

## 🔗 相关功能

- **手动回应功能**：可以使用 `interaction_id` 通过发送消息API进行手动回应
- **交互通知**：禁用转换时的原始通知功能
- **消息过滤**：可以通过 `real_message_type` 字段过滤交互消息
